# 맥점 봇 매매 규칙 (BTCUSDT · 1d·1h·15m · 양방향)

> 이 문서는 사용자와 합의해 확립한 **신호 규칙의 사람 읽는 원본**이다.
> 코드(strategy.py 등)가 구현, 이 문서가 설명·복구용 기준.
> 최종 갱신: 2026-07-02. 롱 기준 서술, **숏은 모두 거울(반대)**.

---

## 1. 신호 구조
- **필수 2개(둘 다 충족)** + **나머지 6개 중 ≥3개** → 신호 성립 (2026-07-05: 대각선 제외로 풀 7→6, 문턱 4→3)
- 롱·숏 완전 대칭
- 잠정(봉 형성중, 되돌리면 취소 가능)·확정(봉 마감) **조건 동일, 타이밍만 다름**

## 2. 필수 2 (반드시)
1. **종가 > 선행스팬1** (숏: <)
   - 선행스팬1 = **(전환선+기준선)/2 의 현재 끝점**(shift 없음, 차트 초록선 끝). `ich["senkou1"]`(shift26 과거값) 쓰면 안 됨.
2. **종가 > 20일선** (숏: <)

## 3. 나머지 6 (≥3 필요)
1. **후행스팬 > 26봉 전 종가** (숏: <)
2. **전환선 > 기준선** 골든크로스 (숏: 데드크로스)
3. ~~하락/상승 대각선~~ → **2026-07-05 체크리스트 제외.** 자동 추세선(피벗 앵커)이 가파른 랠리·급반전을 못 따라가는 구조적 한계 → **대각선은 사용자가 직접 작도·확인**(카드 📐 문구가 리마인드).
4. **MACD = 방향**: 골든크로스(MACD선 > 시그널선)면 롱 / 데드크로스면 숏. **0선 돌파 불필요.**
5. **스토캐스틱 = 진입시점**: 50선 돌파.
   - 롱 = **50~80** + %K>%D(GC) + 상향 / 숏 = **20~50** + %K<%D(DC) + 하향
   - **과열(80↑)이면 상방이어도 롱 ❌**(하락 위험), **침체(20↓)면 하방이어도 숏 ❌**(상승 위험)
   - 설정: **키움식 12/5/5** (%K길이12·%K스무딩5·%D스무딩5)
6. **RCI 보조**: **단기선(9) > 중기선(13) 골든크로스 + 단기선 0선 위 + 상향** (숏: 단<중 DC + 단 0선 아래 + 하향)
   - 각도 검증: 0선 위라도 단기선이 꺾여 내려오는 중이면 무효(오실레이터 방향 원칙)
   - 장기선(26)의 0선 돌파는 '롱 유지'일 뿐 진입엔 늦어서 진입 판정엔 안 씀
7. **RCI 그린(장기26) 0선 위** (숏: 아래)
   - 늦지만 추세확인 보조. 이게 같이 켜지면 더 확실(베스트 배지 조건)

> 핵심 역할: **MACD=방향 / 스토=진입시점 / RCI(단9·중13)=보조 / 그린26=확인**

## 4. 오실레이터 일반 원칙 (위치 + 방향)
- 선(0선/50선)의 **위/아래가 '편'을 정하고**, 크로스·각도가 **유효화/무효화**한다.
- 선 위라도 꺾이거나(DC/하향) 반대크로스면 → 롱 ❌ (단, 아직 선 위면 숏도 아님 = 중립)
- 지표 설정: 스토 12/5/5, MACD 12/26/9 EMA, RCI 9/13/26, 일목 9/26/52

## 5. 손절·익절·최종 판단
- **손절**: 직전저점(롱)/직전고점(숏). 불명확하면 ATR(×2) 대체.
- **익절**: **직접 판단**(카드에 익절 라인 없음).
- **진입 전 가로 매물대·채널·피보나치 반드시 작도 후 최종 결정** — 봇은 후보 알림, 최종 판단은 사람(카드에 📐 강조문구 표시).

## 7. 신호 발송 규칙 (노이즈 억제)
- 같은 봉·같은 방향은 **1회만**
- **직전 발송과 같은 방향이 연속이면 억제**(봉을 넘어 유지). **반대 신호(변곡)가 나와야만 재허용 — 시간 리셋 없음**(2026-07-04).
- **예외(2026-07-06): 🎯 막돌파 맥점 / ⚡ 급반전 후보는 같은 방향이어도 발송** — 단 "직전 봉엔 없다가 막 켜진" 에지에서만 1회(상태 유지 중 반복 발송 없음, 같은 봉 잠정→확정 중복도 차단).
- 잠정은 마감 전 일정 시간(1h=15분/15m=5분) 이상 남았을 때만, 필수2 충족 시에만 발송.
- 정확성 > 빈도. 신호 자주 없어도 됨.
