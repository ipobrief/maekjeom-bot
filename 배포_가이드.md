# 맥점 알림봇 — GitHub Actions 클라우드 배포 가이드

PC를 꺼도 15분마다 깃허브가 신호를 점검해 텔레그램으로 알림. **무료, 서버 관리 없음.**

## 0. 준비
- GitHub 계정 (없으면 github.com 에서 가입)
- 텔레그램 봇 토큰 + chat_id (이미 발급 완료)

## 1. 저장소(repository) 만들기
1. github.com 로그인 → 오른쪽 위 `+` → **New repository**
2. 이름: `maekjeom-bot` (아무거나)
3. **Public** 선택 (무료 무제한. 비밀정보는 코드에 없음)
4. **Create repository**

## 2. 코드 올리기
필요한 파일: `alert_bot.py`, `data.py`, `strategy.py`, `indicators.py`, `requirements.txt`
(backtest.py / sweep.py 는 선택)

1. 저장소 화면 → **Add file → Upload files**
2. 위 파일들을 드래그해서 올리고 → **Commit changes**
3. 워크플로 파일은 따로: **Add file → Create new file**
4. 파일명 칸에 정확히 입력: `.github/workflows/alert.yml`
   (슬래시 `/` 를 치면 폴더가 자동 생성됨)
5. 내용에 `alert.yml` 전체를 붙여넣기 → **Commit changes**

## 3. 비밀정보(Secrets) 등록 — 토큰은 여기에만
1. 저장소 → **Settings** → 왼쪽 **Secrets and variables → Actions**
2. **New repository secret**
   - Name: `TELEGRAM_TOKEN` / Secret: (봇 토큰) → Add
3. 다시 **New repository secret**
   - Name: `TELEGRAM_CHAT_ID` / Secret: (chat_id 숫자) → Add

## 4. 작동 테스트
1. 저장소 → **Actions** 탭
2. 왼쪽에서 **맥점 알림봇 (BTC 15분)** 클릭
3. 오른쪽 **Run workflow** 버튼 → 실행
4. 잠시 후 초록색 체크 → 신호가 있으면 텔레그램 도착

## 동작 방식
- 15분마다 자동 실행 (`*/15 * * * *`, UTC) — 깃허브 부하로 몇 분 지연될 수 있음
- 방금 마감된 봉(신선도 16분 이내)에 신호가 있을 때만 텔레그램 전송 → 중복 알림 방지
- 신호 없으면 조용함 (로그만 남고 알림 안 옴)

## 주의
- 스케줄 워크플로는 저장소가 60일간 활동 없으면 깃허브가 자동 비활성화함 → 가끔 커밋하거나 Run workflow 눌러주면 유지
- 전략 변경은 PC에서 코드 고친 뒤 다시 Upload(또는 git push)하면 반영됨
